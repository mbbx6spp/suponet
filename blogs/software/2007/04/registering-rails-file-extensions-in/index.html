Registering Rails File Extensions in GEdit

2007-04-30 20:59:00 -0500
I just wanted to slap a quick howto for a couple of my professional and open source associates to get going with GEdit quickly while developing Ruby on Rails applications. GEdit uses Gnome's VFS for mime-type determination. To register new file mime-types for Gnome's VF do the following: %ol %li Create XML file with the following contents: %pre :preserve <?xml version="1.0" encoding="UTF-8"?> <mime-info xmlns="http://www.freedesktop.org/standards/shared-mime-info"> <mime-type type="application/x-ruby"> <alias type="application/ruby" /> <comment xml:lang="en">Ruby Rake Tasks</comment> <glob pattern="*.rake"/> </mime-type> <mime-type type="application/x-ruby"> <comment xml:lang="en">Rakefile</comment&gt <glob pattern="Rakefile"/> </mime-type> <mime-type type="text/html"> <comment xml:lang="en">eRB HTML Markup</comment> <glob pattern="*.rhtml"/> </mime-type> <mime-type type="application/x-javascript"> <comment xml:lang="en">eRB JavaScript</comment> <glob pattern="*.rjs"/> </mime-type> <mime-type type="text/xml"> <comment xml:lang="en">eRB XML</comment> <glob pattern="*.rxml"/> </mime-type> <mime-type type="text/x-yaml"> <comment xml:lang="en">YAML</comment> <glob pattern="*.yml"/> </mime-type> </mime-info> %li Copy file to %code /usr/share/mime/packages/rails.xml %li Execute %code sudo update-mime-database /usr/share/mime %li Enjoy!
