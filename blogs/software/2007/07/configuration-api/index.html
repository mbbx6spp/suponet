Twitter4R v0.2.0: Configuration API

2007-07-08 14:48:00 -0500
The %b Configuration API allows applications using Twitter4R configure static connection options such as: %ul %li connection protocol (e.g. HTTP and SSL) %li host name and port (good for testing or if the Twitter REST API moved to a different subdomain) %li proxy settings (i.e. proxy host, port, username and password) %li user agent identifier %li %tt X-Twitter-Client* HTTP headers for Twitter's internal usage analysis This API intentionally tries to look like the Rails initializer/configuration API: %pre %code require('rubygems') gem('twitter4r', '0.2.0') require('twitter') Twitter::Client.configure do |conf| \# We can set Twitter4R to use %tt :ssl or %tt :http to connect to the Twitter API. \# Defaults to %tt :ssl conf.protocol = :ssl \# We can set Twitter4R to use another host name (perhaps for internal \# testing purposes). \# Defaults to 'twitter.com' conf.host = 'twitter.com' \# We can set Twitter4R to use another port (also for internal \# testing purposes). \# Defaults to 443 conf.port = 443 \# We can set proxy information for Twitter4R \# By default all following values are set to = succeed "." do %tt nil conf.proxy_host = 'myproxy.host' conf.proxy_port = 8080 conf.proxy_user = 'myuser' conf.proxy_pass = 'mypass' \# We can also change the User-Agent and X-Twitter-Client* HTTP headers conf.user_agent = 'MyAppAgentName' conf.application_name = 'MyAppName' conf.application_version = 'v1.5.6' conf.application_url = 'http://myapp.url' end If using %a{:href => "http://twitter4r.rubyforge.org"} Twitter4R %span{:style => "display: none"} (open source Ruby library for Twitter REST API) in a Rails application I would recommend adding the necessary Twitter4R configuration code in %tt config/twitter.rb then including it in the %tt config/environment.rb file to keep configurations separate. %h4 Upgrading from 0.1.1 In Twitter4R v0.1.1 you could configure protocol, server host, server port and proxy settings using the following code: %pre :preserve Twitter::Client.conf( :ssl => true, :port => 443, :proxy_host => 'myproxy.host', :proxy_port => 8080) This API no longer exists, so if upgrading to Twitter4R 0.1.1 you will need to change the above to something like the following: %pre :preserve Twitter::Client.configure do |conf| conf.protocol = :ssl conf.port = 443 conf.proxy_host = 'myproxy.host' conf.proxy_port = 8080 end This change shouldn't be too painful for people I hope. Please refer to the %a{:href => "http://twitter4r.rubyforge.org/releases/v0.2.0/rdoc/"} Twitter4R v0.2.0 RDoc for more information.
