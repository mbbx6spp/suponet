HTML5: Link Relations

%p When viewing the HTML source on a number of large and high traffic sites (mostly blogs) I find a number of link relations that either use non-existant relations, misspell or deviate from the specified intent. Worst of all are sites that never use link relations when doing so would offer their audience a richer, more meaningful experience of the content. %p By ensuring standards compliance, browsers and other tools used to render web content could offer more useful and intuitive user interfaces for the content on sites. %p In an effort to promote proper use (as in intended by the specification) of these link relations in HTML5, I decided to research (informally) the major offenders in the wild and ways they could be fixed. %p The first order of business is to establish exactly what are the purposes of link relations and where should they be defined. %h3 What on earth is a link relation? %p Also called a %em link type, a link relation is a way to explain why a link points to another resource. Is it augmenting the content in the current document in some way or is it simply linking to another resource for extra information specific to the content. %p Some reasons why an web author, editor or publisher might want to link to other resources are to: %ul %li point to style rules for the page %li offer alternate versions (formats) of the current page %li offer translations of the current page %li point to the next chapter of a book or next post in a blog series %li point to an archives listing %li offer a way to subscribe to content, like offering an RSS or Atom feed for a blog or podcast. %p These are by no means the only reasons to link to other resources outside of the page using relations. %h3 How are relations defined? %p The way to define relationships between resources (pages) is to use the rel attribute on link, a or even area elements. For example, <link rel="license" href="http://creativecommons.org/licenses/by-nc-sa/3.0/"> creates a link indicating that the content at that resource is covered by the copyright license described by the referenced document (in this case the Attribution- NonCommercial-ShareAlike 3.0 Unported license). %h3 What relations are standard? %p Suppose we have a blog with URL of http://domain.tld/blogs/blog-title for the home page of a blog of mine. The following are standard link relations that might be used. %ul %li %strong alternate %span e.g. in the head element point to Atom feed as an alternative view of the blog %code{:class => "language-html"} <link rel="alternate" href="http://domain.tld/blogs/blog-title/feed.atom"> %li %strong archives %span e.g. in a side menu in the body of the page you might link to the December 2010 archive %code{:class => "language-html"} <a rel="archives" href="http://domain.tld/blogs/blog-title/2010/12/archive.html"> %li %strong author %span e.g. in the head element point to author's home page %code{:class => "language-html"} <link rel="author" href="http://susanpotter.net"> %li %strong bookmark %span e.g. in the body in a link element (e.g. a element) put permalink for bookmarking %code{:class => "language-html"} <a rel="bookmark" href="http://domain.tld/blogs/blog-title"> %li %strong external %span e.g. in the body in a link element we might link to a resource on an external site %code{:class => "language-html"} <a rel="external" href="http://anothersite.com/some-uri"> %li %strong first %li %strong help %span e.g. in the head of the page we can add a link to a help page or in the body put an anchor to a context-specific help page %code{:class => "language-html"} <a rel="help" href="http://domain.tld/help/blog.html"> %li %strong icon %span e.g. in the head of the page we can add an icon that represents the page %code{:class => "language-html"} <link rel="icon" href="http://domain.tld/icons/blog-icon.png"> %li %strong index %li %strong last %li %strong license %li %strong next %li %strong nofollow %li %strong noreferrer %li %strong pingback %li %strong prefetch %li %strong prev %li %strong search %li %strong stylesheet %li %strong sidebar %li %strong tag %li %strong up
